# Практическое занятие 5: Задача регрессии — прогнозирование стоимости домов

## Суть задания

Задача **регрессии** (не классификации) — предсказание непрерывного числового значения. Нужно по характеристикам района оценить медианную стоимость дома. Датасет California Housing: 20,640 районов Калифорнии, 9 числовых + 1 категориальный признак. Целевая переменная — `median_house_value`.

---

## Теоретическая база

### Регрессия vs классификация

- **Классификация** — предсказываем метку класса (ушёл/остался, сорт винограда)
- **Регрессия** — предсказываем число (цена дома, температура, зарплата)

Метрики разные: для классификации — accuracy, F1; для регрессии — MAE, MSE, R2.

### Метрики регрессии

**MAE (Mean Absolute Error)** — средняя абсолютная ошибка:
MAE = (1/n) * sum(|y_real - y_pred|)

Интуитивно: "в среднем модель ошибается на X долларов". Легко интерпретировать. Не штрафует выбросы.

**MSE (Mean Squared Error)** — средняя квадратичная ошибка:
MSE = (1/n) * sum((y_real - y_pred)^2)

Возводит ошибку в квадрат — большие ошибки штрафуются непропорционально сильно. Полезно, если крупные промахи критичны.

**RMSE (Root MSE)** — корень из MSE. Возвращает ошибку в исходные единицы ($), удобнее для интерпретации, чем MSE.

**R2 (коэффициент детерминации)**:
R2 = 1 - (sum((y_real - y_pred)^2) / sum((y_real - y_mean)^2))

Показывает, какую долю дисперсии целевой переменной объясняет модель:
- R2 = 1.0 — модель идеально предсказывает
- R2 = 0.0 — модель не лучше, чем просто предсказывать среднее
- R2 < 0 — модель хуже среднего (бывает при совсем плохих моделях)

### LinearRegression

Ищет линейную зависимость: y = b0 + b1*x1 + b2*x2 + ... + bn*xn

- `coef_` (b1, b2, ..., bn) — коэффициенты при признаках. Показывают, на сколько изменится предсказание при увеличении признака на 1.
- `intercept_` (b0) — свободный член. Предсказание при всех признаках = 0.

Плюсы: быстрая, интерпретируемая (можно объяснить бизнесу). Минусы: ловит только линейные зависимости.

### DecisionTreeRegressor

Дерево решений для регрессии. Разбивает пространство признаков на прямоугольные области и в каждой предсказывает среднее значение целевой переменной.

`max_depth=3` — дерево может сделать максимум 3 уровня разбиений. Это ограничение предотвращает переобучение, но уменьшает гибкость модели.

Плюсы: ловит нелинейности. Минусы: с max_depth=3 очень ограничена (всего 2^3=8 листьев = 8 разных предсказаний).

### Особенность датасета: потолок $500,001

В California Housing значения обрезаны: дома дороже $500,001 записаны как ровно $500,001. Это около 5% выборки. Модель не может правильно предсказать стоимость для дорогих районов — это вносит систематическую ошибку.

## Что спросит преподаватель (вероятные вопросы)

1. **Чем регрессия отличается от классификации?** — Регрессия предсказывает непрерывное число (цена), классификация — дискретный класс (ушёл/остался). Метрики разные: MAE/MSE/R2 вместо accuracy/F1.

2. **Что такое R2 и что значит 0.51?** — Доля дисперсии целевой переменной, объяснённая моделью. 0.51 = модель объясняет 51% разброса цен. Оставшиеся 49% — факторы, не учтённые моделью (локация, состояние дома и т.д.).

3. **Почему только 3 признака?** — По заданию: "выберите 2-3 наиболее понятных признака для наглядности". С большим числом признаков R2 был бы выше, но цель — интерпретируемость.

4. **Как интерпретировать коэффициент при median_income?** — При увеличении медианного дохода в районе на $10,000, модель предсказывает рост стоимости дома на ~$44,231. Это самый сильный фактор.

5. **Почему коэффициент rooms_per_household отрицательный?** — Контринтуитивно, но при фиксированных остальных признаках (доход, возраст) больше комнат часто означает удалённый пригород с низкими ценами. Это эффект confounding — без контроля за локацией коэффициент может быть обратным.

6. **Почему дерево хуже линейной модели?** — max_depth=3 сильно ограничивает дерево (всего 8 листьев = 8 разных предсказаний). Линейная модель имеет непрерывный выход. С max_depth=10+ дерево обычно побеждает, но склонно к переобучению.

7. **Что за пик на $500,001 в гистограмме?** — Искусственный потолок (цензурирование). Все дома дороже $500k записаны как ровно $500,001. Это ~5% данных. Модель не может корректно предсказывать в этом диапазоне.

8. **Зачем RMSE, если есть MAE?** — MAE — средняя ошибка в абсолютных единицах. RMSE штрафует крупные ошибки сильнее (из-за квадрата). Если важно минимизировать "катастрофические" промахи — смотрим на RMSE.

9. **Что такое intercept?** — Свободный член линейной модели. Формально — предсказание при всех признаках = 0. На практике часто не имеет физического смысла (доход 0, возраст 0, комнат 0 — таких районов не бывает).

10. **Как scatter plot показывает качество модели?** — Чем ближе точки к диагонали y=x, тем точнее предсказания. В нашем случае облако размыто — ошибки значительны. Для дорогих домов (>$400k) точки ниже диагонали — модель систематически занижает.
