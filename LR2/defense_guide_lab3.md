## Ответы на возможные вопросы

### Q: Как работает механизм вывода?

**A:** Используется библиотека experta:
1. Декларируем факты через `declare()`
2. Движок автоматически проверяет условия всех правил
3. Если условия выполнены → правило срабатывает
4. Правило добавляет новые факты (диагноз, рекомендации)
5. Новые факты могут активировать другие правила (цепочка)

**Пример цепочки:**
```
Симптомы → [R001] Грипп → [R008] Высокий риск у пожилых
```

---

### Q: Почему выбрали именно эти 7 правил?

**A:** Они образуют законченные цепочки вывода:
- **R001, R002, R003, R006** - базовая диагностика (4 разных диагноза)
- **R005** - выявление осложнений
- **R012** - критические ситуации (безопасность!)
- **R008** - вторичный вывод (демонстрирует цепочку)

---

### Q: Как реализован блок объяснений?

**A:** Каждое правило при срабатывании вызывает:

```python
self.add_explanation('R001', 'Название', 'Обоснование')
```

Это добавляет запись в список `fired_rules[]`:
- ID правила
- Название
- Обоснование (почему сработало)
- Время срабатывания

Затем список выводится в конце работы.

---

### Q: Что такое фактор уверенности (CF)?

**A:** Это числовое значение от 0 до 1, показывающее уверенность системы:
- **1.0** - абсолютная уверенность (критические состояния)
- **0.85-0.95** - высокая (специфические симптомы)
- **0.70-0.80** - умеренная (нужно подтверждение)

В нашей системе CF хранится в факте Diagnosis:
```python
Diagnosis(name='Грипп типа A', confidence=0.85)
```

### Q: Какие были проблемы при тестировании?

**A:** Нашли 2 проблемы:

**Проблема 1:** R005 (Пневмония) не срабатывало вместе с R012 (Критическое)

**Причина:** Условие `NOT(Diagnosis())` блокировало

**Решение:** Убрали `NOT(Diagnosis())` из R005 - пневмония может быть ПРИЧИНОЙ критического состояния

**Проблема 2:** Рекомендации выводились в случайном порядке

**Решение:** Добавили сортировку по приоритету (экстренный → критический → высокий → средний → низкий)
