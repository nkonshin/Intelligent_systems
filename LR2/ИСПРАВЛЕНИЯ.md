## Проблемы, обнаруженные при проверке

### 1. Проблема совместимости с Python 3.10+

**Проблема:**
```
AttributeError: module 'collections' has no attribute 'Mapping'
```

**Причина:**  
Библиотека `experta==1.9.4` использует `frozendict==1.2`, который обращается к `collections.Mapping`. В Python 3.10+ эти классы были перемещены в `collections.abc`.

**Решение:**  
Добавлен патч совместимости в начало файла `medical_expert_system.py`:

```python
# Патч для совместимости с Python 3.10+
import sys
import collections
import collections.abc
for name in ['Mapping', 'MutableMapping', 'Iterable', 'MutableSet', 'Callable']:
    if not hasattr(collections, name):
        setattr(collections, name, getattr(collections.abc, name))
```

**Статус:** ✅ ИСПРАВЛЕНО

### 2. Ошибка типов в предикатах

**Проблема:**
```
TypeError: '>' not supported between instances of 'str' and 'int'
```

**Причина:**  
Предикаты в правилах пытались сравнивать значения без проверки типа. Когда experta проверял правила на всех фактах, он мог попытаться сравнить строковое значение с числом.

**Решение:**  
Добавлена проверка типов во все числовые предикаты:

**Было:**
```python
Symptom(name='температура', value=P(lambda x: x > 38.5))
```

**Стало:**
```python
Symptom(name='температура', value=P(lambda x: isinstance(x, (int, float)) and x > 38.5))
```

**Затронутые правила:**
- R001 (Грипп типа A)
- R002 (ОРВИ)
- R003 (COVID-19)
- R006 (Легкая простуда)
- R005 (Пневмония)
- R012 (Критическое состояние)
- R008 (Группа риска)

**Статус:** ✅ ИСПРАВЛЕНО

### 3. Изменение оригинального словаря в load_scenario

**Проблема:**  
Функция `load_scenario` использовала `fact_data.pop('type')`, что изменяло оригинальный словарь из JSON.

**Решение:**  
Создается копия словаря перед извлечением типа:

**Было:**
```python
for fact_data in data.get('facts', []):
    fact_type = fact_data.pop('type')
```

**Стало:**
```python
for fact_data in data.get('facts', []):
    fact_dict = dict(fact_data)
    fact_type = fact_dict.pop('type')
```

**Статус:** ✅ ИСПРАВЛЕНО

### 4. Дублирование срабатывания правила R012

**Наблюдение:**  
В сценарии #3 правило R012 срабатывает дважды, потому что есть два критических симптома:
- Тяжелая одышка
- Низкая сатурация O₂ (88%)

**Анализ:**  
Это не ошибка, а особенность работы experta. Правило срабатывает для каждого условия в OR, которое выполняется.

**Решение:**  
Можно добавить проверку `NOT(Diagnosis(name='КРИТИЧЕСКОЕ СОСТОЯНИЕ'))` в правило R012, чтобы оно срабатывало только один раз.

**Статус:** ⚠️ ОПЦИОНАЛЬНО (не критично)
