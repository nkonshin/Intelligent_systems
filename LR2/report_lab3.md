### 1.3. Определение классов Fact

Разработано **6 классов фактов**:

```python
class Patient(Fact):      # Данные пациента (возраст, хронические болезни)

class FiredRule(Fact):    # Сработавшие правила (для объяснений)
```

## ЭТАП 2: РЕАЛИЗАЦИЯ ЯДРА ЭС

### 2.1. Реализованные правила

#### Правило R001: Диагностика гриппа типа A

```python
@Rule(
    Symptom(name='температура', value=P(lambda x: x > 38.5)),
    Symptom(name='начало', value='резкое'),
    Symptom(name='ломота_в_мышцах', value='высокая'),
    Symptom(name='слабость', value='выраженная'),
    NOT(Diagnosis())
)
def rule_001_flu_type_a(self):
    """R001: Определение гриппа типа A (CF: 0.85)"""
    self.declare(Diagnosis(
        name='Грипп типа A',
        confidence=0.85,
        severity='средняя'
    ))
    self.declare(Recommendation(...))
    self.add_explanation('R001', 'Диагностика гриппа типа A', ...)
```

**Особенности:**
- Использует предикат `P(lambda x: x > 38.5)` для проверки температуры
- Проверяет отсутствие диагноза `NOT(Diagnosis())` во избежание конфликтов
- Автоматически добавляет объяснение при срабатывании

#### Правило R003: COVID-19

```python
@Rule(
    OR(
        Symptom(name='потеря_обоняния', value=True),
        Symptom(name='потеря_вкуса', value=True)
    ),
    Symptom(name='температура', value=P(lambda x: x > 37.5)),
    OR(
        Symptom(name='кашель', value='сухой'),
        Symptom(name='одышка', value=MATCH.any_value)
    ),
    NOT(Diagnosis())
)
```

**Особенности:**
- Использует оператор `OR` для альтернативных условий
- `MATCH.any_value` - срабатывает при любом значении одышки

#### Правило R012: Критическая ситуация

```python
@Rule(
    OR(
        Symptom(name='одышка', value='тяжелая'),
        Symptom(name='сатурация_O2', value=P(lambda x: x < 92)),
        AND(
            Symptom(name='температура', value=P(lambda x: x > 40)),
            Symptom(name='жаропонижающие_помогают', value=False)
        ),
        Symptom(name='спутанность_сознания', value=True)
    )
)
```

**Особенности:**
- Наивысший CF = 1.0 (абсолютная уверенность)
- НЕТ проверки `NOT(Diagnosis())` - срабатывает независимо
- Комбинация `OR` и `AND` для сложных условий

#### Правило R008: Группа риска

```python
@Rule(
    Patient(age=P(lambda x: x > 65)),
    OR(
        Diagnosis(name='Грипп типа A'),
        Diagnosis(name='ОРВИ')
    ),
    Patient(chronic_diseases=True)
)
```

**Особенности:**
- **Вторичное правило** - зависит от срабатывания R001 или R002
- Демонстрирует цепочку вывода: Симптомы → Диагноз → Уровень риска

### 2.2. Механизм объяснений

Реализован блок объяснений через:

1. **Список сработавших правил:**
```python
self.fired_rules = []
```

2. **Метод добавления объяснения:**
```python
def add_explanation(self, rule_id, rule_name, reasoning):
    self.fired_rules.append({
        'id': rule_id,
        'name': rule_name,
        'reasoning': reasoning,
        'timestamp': datetime.now().strftime('%H:%M:%S')
    })
```

3. **Вывод объяснений:**
```python
def print_results(self):
    # ... вывод диагнозов и рекомендаций ...
    
    if self.fired_rules:
        print(f"\nСработало правил: {len(self.fired_rules)}\n")
        for i, rule in enumerate(self.fired_rules, 1):
            print(f"{i}. [{rule['id']}] {rule['name']}")
            print(f"   Обоснование: {rule['reasoning']}")
```

### 3.4. ИСПРАВЛЕНИЕ №1: Порядок срабатывания правил

**Проблема:** R012 срабатывает раньше R005 и блокирует его через `NOT(Diagnosis())`

**Решение:** Убрать `NOT(Diagnosis())` из правила R005, так как пневмония - это осложнение, которое может сосуществовать с критическим состоянием.

**Код ДО исправления:**
```python
@Rule(
    Symptom(name='длительность_заболевания', value=P(lambda x: x > 7)),
    Symptom(name='температура', value=P(lambda x: x > 38)),
    Symptom(name='кашель_с_мокротой', value=True),
    Symptom(name='боль_в_груди', value=True),
    Symptom(name='одышка', value=MATCH.any_value),
    NOT(Diagnosis())  # ← ПРОБЛЕМА: блокирует срабатывание
)
def rule_005_pneumonia(self):
```

**Код ПОСЛЕ исправления:**
```python
@Rule(
    Symptom(name='длительность_заболевания', value=P(lambda x: x > 7)),
    Symptom(name='температура', value=P(lambda x: x > 38)),
    Symptom(name='кашель_с_мокротой', value=True),
    Symptom(name='боль_в_груди', value=True),
    Symptom(name='одышка', value=MATCH.any_value)
    # NOT(Diagnosis()) УДАЛЕНО - позволяет сосуществовать с R012
)
def rule_005_pneumonia(self):
```

**Обоснование:**
- Пневмония и критическое состояние - не взаимоисключающие диагнозы
- В реальности пневмония часто ЯВЛЯЕТСЯ причиной критического состояния
- Оба диагноза должны быть установлены для полной картины

#### Результат ПОСЛЕ исправления:

### 3.5. ИСПРАВЛЕНИЕ №2: Улучшение вывода результатов

**Проблема:** Рекомендации выводились в случайном порядке

**Решение:** Добавить сортировку рекомендаций по приоритету

**Код ДО:**
```python
for rec in recommendations:
    print(f"  • {rec.get('text', '')}")
```

**Код ПОСЛЕ:**
```python
priority_order = {
    'экстренный': 0, 
    'критический': 1, 
    'высокий': 2, 
    'средний': 3, 
    'низкий': 4
}
sorted_recs = sorted(
    recommendations, 
    key=lambda x: priority_order.get(x.get('priority', 'средний'), 3)
)

for rec in sorted_recs:
    # ... вывод с эмодзи в зависимости от приоритета
```

**Результат:** Рекомендации теперь выводятся от наиболее срочных к менее срочным.
