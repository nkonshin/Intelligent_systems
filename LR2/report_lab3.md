## 2. ЦЕЛИ СИСТЕМЫ

Основные цели разработки экспертной системы:

1. **Автоматизированная первичная диагностика** - быстрое определение наиболее вероятного диагноза на основе введенных симптомов

2. **Поддержка принятия решений** - предоставление обоснованных рекомендаций по необходимости обращения к врачу

3. **Выявление критических состояний** - немедленное распознавание опасных симптомов, требующих экстренной медицинской помощи

4. **Прозрачность диагностики** - объяснение логики рассуждений системы для понимания пользователем

5. **Снижение нагрузки на медицинские учреждения** - фильтрация случаев, не требующих немедленного обращения к врачу

---

## 3. ЦЕЛЕВАЯ АУДИТОРИЯ

**Основные пользователи:**

- **Пациенты** - люди с симптомами простудных заболеваний, нуждающиеся в первичной консультации
- **Возрастные группы**: взрослые (18-65 лет), пожилые (65+), родители детей до 18 лет
- **Медицинские работники** - врачи общей практики, фельдшеры для поддержки принятия решений
- **Операторы телемедицины** - специалисты call-центров медицинских учреждений

**Требования к пользователям:**
- Базовые навыки работы с компьютером
- Умение описать свои симптомы
- Понимание, что система не заменяет профессиональную медицинскую помощь

---

## 4. ПРЕДМЕТНАЯ ОБЛАСТЬ

**Область применения:** Диагностика распространенных респираторных заболеваний

**Охватываемые заболевания:**
- Вирусные инфекции: Грипп типа A, Грипп типа B, ОРВИ, COVID-19
- Бактериальные инфекции: Ангина, Бронхит
- Осложнения: Пневмония, Синусит

**Анализируемые симптомы:**
- Респираторные: кашель, насморк, боль в горле, одышка
- Общие: температура, слабость, головная боль, боль в мышцах
- Специфические: потеря обоняния/вкуса, сатурация кислорода

**Факторы риска:**
- Возраст пациента (дети до 3 лет, пожилые старше 65 лет)
- Наличие хронических заболеваний
- Длительность заболевания

**Ограничения предметной области:**
- Система не диагностирует редкие заболевания
- Не заменяет лабораторные исследования
- Требует подтверждения диагноза врачом

## 5. ОСНОВНЫЕ ФУНКЦИИ

**Функция 1: Сбор и анализ симптомов**
- Интерактивный ввод симптомов через консольный интерфейс
- Загрузка тестовых сценариев из JSON-файлов
- Валидация введенных данных

**Функция 2: Диагностика заболеваний**
- Применение продукционных правил к введенным симптомам
- Вычисление факторов уверенности (CF) для каждого диагноза
- Разрешение конфликтов при срабатывании нескольких правил

**Функция 3: Оценка рисков**
- Определение групп риска (пожилые, дети)
- Выявление критических состояний
- Оценка необходимости госпитализации

**Функция 4: Генерация рекомендаций**
- Формирование плана лечения
- Определение приоритетов рекомендаций (экстренный, критический, высокий, средний, низкий)
- Выдача предупреждений о необходимости обращения к врачу

**Функция 5: Объяснение рассуждений**
- Отображение списка сработавших правил
- Обоснование каждого вывода
- Визуализация цепочки рассуждений

## 6. ДЕРЕВО РЕШЕНИЙ

Система использует продукционные правила, которые образуют следующие цепочки принятия решений:

**Цепочка 1: Диагностика вирусных инфекций**
```
Входные симптомы
    ↓
Температура > 38.5°C?
    ├─ ДА → Резкое начало?
    │         ├─ ДА → Ломота в мышцах?
    │         │         ├─ ДА → [R001] Грипп типа A (CF: 0.85)
    │         │         └─ НЕТ → Проверка других правил
    │         └─ НЕТ → [R002] ОРВИ (CF: 0.90)
    └─ НЕТ → Температура < 37.5°C?
              └─ ДА → [R006] Легкая простуда (CF: 0.95)
```

**Цепочка 2: Выявление COVID-19**
```
Потеря обоняния/вкуса?
    ├─ ДА → Температура > 37.5°C?
    │         ├─ ДА → Сухой кашель ИЛИ одышка?
    │         │         └─ ДА → [R003] COVID-19 (CF: 0.75)
    │         └─ НЕТ → Проверка других правил
    └─ НЕТ → Проверка других правил
```

**Цепочка 3: Выявление осложнений**
```
Длительность > 7 дней?
    ├─ ДА → Кашель с мокротой + боль в груди?
    │         └─ ДА → [R005] Пневмония (CF: 0.70)
    └─ НЕТ → Проверка других правил
```

**Цепочка 4: Критические состояния**
```
Одышка тяжелая ИЛИ сатурация < 92%?
    ├─ ДА → [R012] КРИТИЧЕСКОЕ СОСТОЯНИЕ (CF: 1.0)
    │         └─ Немедленная госпитализация
    └─ НЕТ → Температура > 40°C без эффекта от жаропонижающих?
              └─ ДА → [R012] КРИТИЧЕСКОЕ СОСТОЯНИЕ (CF: 1.0)
```

**Цепочка 5: Оценка рисков**
```
Установлен диагноз (Грипп/ОРВИ)?
    └─ ДА → Возраст > 65 лет?
              └─ ДА → Хронические заболевания?
                        └─ ДА → [R008] Высокий риск (CF: 0.88)
```

**Алгоритм работы системы:**
1. Сбор симптомов → 2. Формирование фактов → 3. Активация правил → 4. Разрешение конфликтов → 5. Вывод диагноза → 6. Генерация объяснений → 7. Формирование рекомендаций

## 7. ОГРАНИЧЕНИЯ СИСТЕМЫ

**Технические ограничения:**
- Консольный интерфейс (нет графического интерфейса)
- Реализовано 7 из 12 правил (ядро системы)
- Отсутствует интеграция с медицинскими базами данных
- Нет сохранения истории консультаций

**Функциональные ограничения:**
- Система не проводит лабораторные анализы
- Не учитывает результаты инструментальных исследований (рентген, КТ)
- Ограниченный набор заболеваний (только простудные)
- Не диагностирует редкие заболевания

**Медицинские ограничения:**
- Не заменяет профессиональную медицинскую консультацию

- Не несет юридической ответственности за рекомендации
- Не предназначена для самолечения

**Ограничения по точности:**
- Факторы уверенности основаны на экспертных оценках, а не на статистических данных
- Не учитывает индивидуальные особенности пациента
- Возможны ложноположительные и ложноотрицательные результаты

### 9.2. Продукционные правила (7 реализованных)

**R001: Грипп типа A (CF: 0.85)**
- IF: температура > 38.5°C AND резкое начало AND ломота в мышцах = высокая AND слабость = выраженная
- THEN: диагноз = "Грипп типа A"

**R002: ОРВИ (CF: 0.90)**
- IF: температура 37.5-38°C AND насморк AND боль в горле = умеренная AND постепенное начало
- THEN: диагноз = "ОРВИ"

**R003: COVID-19 (CF: 0.75)**
- IF: (потеря обоняния OR потеря вкуса) AND температура > 37.5°C AND (сухой кашель OR одышка)
- THEN: диагноз = "Подозрение на COVID-19"

**R006: Легкая простуда (CF: 0.95)**
- IF: температура < 37.5°C AND легкий насморк AND легкое недомогание AND НЕТ сильной головной боли
- THEN: диагноз = "Легкая простуда"

**R005: Пневмония (CF: 0.70)**
- IF: длительность > 7 дней AND температура > 38°C AND кашель с мокротой AND боль в груди AND одышка
- THEN: диагноз = "Риск пневмонии"

**R012: Критическое состояние (CF: 1.0)**
- IF: одышка = тяжелая OR сатурация < 92% OR (температура > 40°C AND жаропонижающие не помогают) OR спутанность сознания
- THEN: действие = "СРОЧНАЯ госпитализация"

**R008: Высокий риск у пожилых (CF: 0.88)**
- IF: возраст > 65 лет AND (диагноз = "Грипп" OR "ОРВИ") AND хронические заболевания
- THEN: риск = "Высокий"

### 9.3. Связи между классами (10 типов)

1. **имеет_симптом** (Заболевание → Симптом)

7. **эффективен_против** (Лечение → Заболевание)
8. **имеет_риск** (Пациент → Осложнение)
9. **предрасположен_к** (Пациент → Заболевание)
10. **усиливает** (Симптом → Симптом)

## 10. АРХИТЕКТУРА СИСТЕМЫ

### 10.1. Компоненты системы

**1. Пользовательский интерфейс**
- Консольный ввод симптомов
- Меню выбора режима работы
- Вывод результатов диагностики

**2. Машина вывода (Inference Engine)**
- Класс: `MedicalDiagnosisEngine(KnowledgeEngine)`
- Прямой вывод (forward chaining)
- Механизм сопоставления с образцом (pattern matching)
- Вычисление факторов уверенности

**3. База знаний**
- 7 продукционных правил с декоратором `@Rule`
- Онтология из 4 классов
- Факты о заболеваниях и симптомах

**4. Рабочая память**
- Хранение текущих фактов пациента
- Список активированных правил
- Промежуточные выводы

**5. Модуль объяснений**
- Список `fired_rules` для отслеживания сработавших правил
- Метод `add_explanation()` для добавления обоснований
- Метод `print_results()` для вывода объяснений

**6. Модуль рекомендаций**
- Генерация рекомендаций на основе диагноза
- Сортировка по приоритетам
- Категоризация (лечение, диагностика, госпитализация, экстренная помощь)

### 10.2. Технологический стек

- **Язык программирования:** Python 3.10+
- **Библиотека для ЭС:** experta (продукционные правила)
- **Формат данных:** JSON (тестовые сценарии)
- **Интерфейс:** Консольный (CLI)

### 10.3. Классы фактов

```python

```

### 3.5. ИСПРАВЛЕНИЕ №2: Улучшение вывода результатов

**Проблема:** Рекомендации выводились в случайном порядке

**Код ДО:**

**Код ПОСЛЕ:**

    'экстренный': 0, 
    'критический': 1, 
    'высокий': 2, 
    'средний': 3, 
    'низкий': 4
}
sorted_recs = sorted(
    recommendations, 
    key=lambda x: priority_order.get(x.get('priority', 'средний'), 3)
