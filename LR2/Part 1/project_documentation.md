# ЭКСПЕРТНАЯ СИСТЕМА ДИАГНОСТИКИ ПРОСТУДНЫХ ЗАБОЛЕВАНИЙ
## Практическое занятие №2 - Вариант 3

---

## ВВЕДЕНИЕ

Экспертная система диагностики простудных заболеваний - это интеллектуальная система, предназначенная для автоматизированной диагностики распространенных респираторных заболеваний на основе введенных пользователем симптомов.

**Цель проекта:** Создать экспертную систему, способную:
- Анализировать симптомы пациента
- Устанавливать наиболее вероятный диагноз
- Предоставлять рекомендации по лечению
- Определять необходимость обращения к врачу

## ЭТАП 1: ОНТОЛОГИЯ СИСТЕМЫ

### 1.1. Основные классы (4 класса верхнего уровня)

#### КЛАСС 1: ЗАБОЛЕВАНИЯ

**Описание:** Основной класс, содержащий все виды простудных заболеваний и их осложнения

**Иерархия (4 уровня):**
```
Заболевания
├── Вирусные_инфекции
│   ├── Грипп
│   │   ├── Грипп_типа_A
│   │   └── Грипп_типа_B
│   ├── ОРВИ
│   └── COVID-19
├── Бактериальные_инфекции
│   ├── Ангина
│   └── Бронхит
└── Осложнения
    ├── Пневмония
    └── Синусит
```

**Атрибуты:**
- название: String - наименование заболевания
- код_МКБ: String - код по МКБ-10
- инкубационный_период: Integer - период в днях
- заразность: Boolean - является ли заразным
- степень_тяжести: Enum (легкая, средняя, тяжелая)

#### КЛАСС 2: СИМПТОМЫ

**Описание:** Класс, описывающий все возможные проявления заболеваний

**Иерархия:**
```
Симптомы
├── Респираторные
│   ├── Кашель (сухой/влажный)
│   ├── Насморк
│   ├── Боль_в_горле
│   └── Одышка
├── Общие
│   ├── Температура
│   ├── Слабость
│   ├── Головная_боль
│   └── Боль_в_мышцах
└── Специфические
    ├── Потеря_обоняния
    └── Сыпь
```

**Атрибуты:**
- название: String
- интенсивность: Integer (шкала 1-10)
- длительность: Integer (дни)
- частота_проявления: Float (0-1, вероятность)

#### КЛАСС 3: ЛЕЧЕНИЕ

**Описание:** Класс методов терапии и лечебных мероприятий

**Иерархия:**
```
Лечение
├── Медикаментозное
│   ├── Противовирусные
│   ├── Антибиотики
│   ├── Жаропонижающие
│   └── Симптоматические
└── Немедикаментозное
    ├── Покой
    ├── Обильное_питье
    └── Увлажнение_воздуха
```

**Атрибуты:**
- название: String
- тип: Enum (медикаментозное, немедикаментозное)
- дозировка: String
- длительность_курса: Integer (дни)
- побочные_эффекты: List<String>

#### КЛАСС 4: ПАЦИЕНТ

**Описание:** Класс характеристик пациента, влияющих на диагностику и лечение

**Иерархия:**
```
Пациент
├── Взрослый
│   ├── Молодой (18-40)
│   └── Пожилой (60+)
└── Ребенок
    ├── Младенец (0-3)
    └── Школьник (7-17)
```

**Атрибуты:**
- возраст: Integer
- наличие_хронических_заболеваний: Boolean
- температура_тела: Float
- иммунный_статус: Enum (слабый, нормальный, хороший)

### 1.2. Отношения между классами (10 связей)

1. **имеет_симптом** (Заболевание → Симптом)
   - Описание: связывает заболевание с характерными симптомами
   - Пример: Грипп → Высокая температура, Ломота в теле

2. **требует_лечения** (Заболевание → Лечение)
   - Описание: указывает необходимые методы лечения
   - Пример: Ангина → Антибиотики, Полоскание горла

3. **может_перейти_в** (Заболевание → Осложнение)
   - Описание: показывает возможные осложнения
   - Пример: ОРВИ → Бронхит, Синусит

4. **проявляется_у** (Заболевание → Пациент)
   - Описание: связь заболевания с типом пациента
   - Пример: Грипп → чаще тяжелое течение у пожилых

5. **сопровождается** (Симптом → Симптом)
   - Описание: сопутствующие симптомы
   - Пример: Кашель → Боль в груди

6. **противопоказан_при** (Лечение → Заболевание)
   - Описание: ограничения применения лечения
   - Пример: Антибиотики противопоказаны при вирусных инфекциях

7. **эффективен_против** (Лечение → Заболевание)
   - Описание: эффективность метода лечения
   - Пример: Осельтамивир → Грипп типа A

8. **имеет_риск** (Пациент → Осложнение)
   - Описание: группы риска по осложнениям
   - Пример: Пожилые → высокий риск Пневмонии

9. **предрасположен_к** (Пациент → Заболевание)
   - Описание: предрасположенность категории пациентов
   - Пример: Дети → частые ОРВИ

10. **усиливает** (Симптом → Симптом)
    - Описание: взаимное усиление симптомов
    - Пример: Высокая температура → усиливает слабость

## ЭТАП 2: БАЗА ПРАВИЛ

### 2.1. Правило R001: Диагностика гриппа типа A

**Название:** "Определение гриппа типа A"

**IF (Условия):**
- температура > 38.5°C AND
- резкое_начало = TRUE AND
- ломота_в_мышцах = высокая AND
- головная_боль = сильная AND
- слабость = выраженная

**THEN (Заключение):**
диагноз = "Грипп типа A"

**Фактор уверенности:** 0.85 (85%)

**Рекомендуемое решение:**
- Постельный режим
- Противовирусные препараты (осельтамивир)
- Обильное питье (не менее 2 литров в день)
- Жаропонижающие при температуре >38.5°C
- Контроль состояния в течение 3 дней

---

### 2.2. Правило R002: Диагностика ОРВИ

**Название:** "Определение обычной ОРВИ"

**IF (Условия):**
- температура = 37.5-38°C AND
- насморк = TRUE AND
- боль_в_горле = умеренная AND
- кашель = легкий AND
- постепенное_начало = TRUE

**THEN (Заключение):**
диагноз = "ОРВИ"

**Фактор уверенности:** 0.90 (90%)

**Рекомендуемое решение:**
- Симптоматическое лечение
- Покой и домашний режим
- Теплое питье (чай, морс)
- Промывание носа солевыми растворами
- Выздоровление через 5-7 дней

---

### 2.3. Правило R003: Подозрение на COVID-19

**Название:** "Выявление COVID-19"

**IF (Условия):**
- потеря_обоняния = TRUE OR потеря_вкуса = TRUE AND
- температура > 37.5°C AND
- (сухой_кашель = TRUE OR одышка = TRUE)

**THEN (Заключение):**
диагноз = "Подозрение на COVID-19"

**Фактор уверенности:** 0.75 (75%)

**Рекомендуемое решение:**
- СРОЧНО сдать ПЦР-тест
- Самоизоляция до результатов
- Мониторинг сатурации кислорода (пульсоксиметр)
- При одышке - немедленное обращение к врачу
- Информирование контактных лиц

---

### 2.4. Правило R004: Диагностика ангины

**Название:** "Определение бактериальной ангины"

**IF (Условия):**
- боль_в_горле = очень_сильная AND
- налет_на_миндалинах = TRUE AND
- увеличенные_лимфоузлы = TRUE AND
- температура > 38°C AND
- НЕТ насморка AND
- НЕТ кашля

**THEN (Заключение):**
диагноз = "Ангина (бактериальная)"

**Фактор уверенности:** 0.80 (80%)

**Рекомендуемое решение:**
- Обязательная консультация врача
- Антибиотики (амоксициллин) - курс 7-10 дней
- Полоскание горла антисептиками (хлоргексидин)
- Жаропонижающие по необходимости
- Постельный режим

---

### 2.5. Правило R005: Риск пневмонии

**Название:** "Выявление осложнения - пневмонии"

**IF (Условия):**
- длительность_заболевания > 7_дней AND
- температура > 38°C AND
- кашель_с_мокротой = TRUE AND
- боль_в_груди = TRUE AND
- одышка = TRUE

**THEN (Заключение):**
диагноз = "Риск пневмонии"

**Фактор уверенности:** 0.70 (70%)

**Рекомендуемое решение:**
- СРОЧНОЕ обращение к врачу
- Рентген грудной клетки
- Общий анализ крови
- Вероятна госпитализация
- Антибиотикотерапия широкого спектра

---

### 2.6. Правило R006: Легкая простуда

**Название:** "Неосложненная простуда"

**IF (Условия):**
- температура < 37.5°C AND
- легкий_насморк = TRUE AND
- легкое_недомогание = TRUE AND
- НЕТ сильной_головной_боли

**THEN (Заключение):**
диагноз = "Легкая простуда"

**Фактор уверенности:** 0.95 (95%)

**Рекомендуемое решение:**
- Домашнее лечение
- Витамин C (1000 мг/день)
- Теплое питье
- Полноценный отдых
- Выздоровление через 3-5 дней

---

### 2.7. Правило R007: Острый бронхит

**Название:** "Определение острого бронхита"

**IF (Условия):**
- продуктивный_кашель = TRUE AND
- хрипы_в_легких = TRUE AND
- длительность_кашля > 5_дней AND
- температура = 37-38°C

**THEN (Заключение):**
диагноз = "Острый бронхит"

**Фактор уверенности:** 0.78 (78%)

**Рекомендуемое решение:**
- Муколитики (АЦЦ, Амброксол)
- Отхаркивающие средства
- Обильное теплое питье
- При бактериальной природе - антибиотики
- Дыхательная гимнастика

---

### 2.8. Правило R008: Группа риска - пожилые

**Название:** "Высокий риск осложнений у пожилых"

**IF (Условия):**
- возраст > 65_лет AND
- (диагноз = "Грипп" OR диагноз = "ОРВИ") AND
- хронические_заболевания = TRUE

**THEN (Заключение):**
риск_осложнений = "Высокий"

**Фактор уверенности:** 0.88 (88%)

**Рекомендуемое решение:**
- Обязательная консультация врача
- Ежедневный мониторинг состояния
- Рассмотреть противовирусную терапию
- Контроль хронических заболеваний
- При ухудшении - госпитализация

---

### 2.9. Правило R009: Группа риска - дети

**Название:** "Особенности течения у детей до 3 лет"

**IF (Условия):**
- возраст < 3_лет AND
- температура > 38.5°C AND
- отказ_от_еды = TRUE

**THEN (Заключение):**
действие = "Требуется осмотр педиатра"

**Фактор уверенности:** 0.92 (92%)

**Рекомендуемое решение:**
- Вызов педиатра на дом
- Контроль температуры каждые 2 часа
- Жаропонижающие по возрасту (парацетамол, ибупрофен)
- Обильное питье
- Наблюдение за дыханием

---

### 2.10. Правило R010: Синусит

**Название:** "Выявление синусита как осложнения"

**IF (Условия):**
- насморк > 10_дней AND
- боль_в_области_пазух = TRUE AND
- гнойные_выделения = TRUE AND
- головная_боль_усиливается_при_наклоне = TRUE

**THEN (Заключение):**
диагноз = "Синусит"

**Фактор уверенности:** 0.82 (82%)

**Рекомендуемое решение:**
- Консультация ЛОР-врача
- Антибиотики (амоксициллин+клавулановая кислота)
- Сосудосуживающие капли (не более 5 дней)
- Промывание пазух ("кукушка")
- Физиотерапия

---

### 2.11. Правило R011: Необходимость антибиотиков

**Название:** "Показания к антибиотикотерапии"

**IF (Условия):**
- (диагноз = "Ангина" OR диагноз = "Синусит" OR диагноз = "Бронхит") AND
- бактериальная_природа = вероятна AND
- длительность > 5_дней

**THEN (Заключение):**
лечение = "Антибиотики показаны"

**Фактор уверенности:** 0.75 (75%)

**Рекомендуемое решение:**
- Назначение антибиотиков ТОЛЬКО врачом
- Препараты выбора: амоксициллин, азитромицин
- Полный курс 5-7 дней (не прерывать!)
- Пробиотики для защиты микрофлоры
- Контроль эффективности через 48-72 часа

---

### 2.12. Правило R012: КРИТИЧЕСКАЯ СИТУАЦИЯ

**Название:** "Критические симптомы - немедленная госпитализация"

**IF (Условия):**
- (одышка = тяжелая OR сатурация_O2 < 92%) OR
- (температура > 40°C AND не_снижается_жаропонижающими) OR
- спутанность_сознания = TRUE

**THEN (Заключение):**
действие = "СРОЧНАЯ госпитализация"

**Фактор уверенности:** 1.0 (100%)

**Рекомендуемое решение:**
- НЕМЕДЛЕННЫЙ вызов скорой помощи (103)
- Транспортировка в стационар
- Интенсивная терапия
- Возможна ИВЛ
- Реанимационные мероприятия по показаниям

## ЭТАП 3: АРХИТЕКТУРА СИСТЕМЫ

### 3.1. Компоненты системы

#### 1. ПОЛЬЗОВАТЕЛЬСКИЙ ИНТЕРФЕЙС

**Функции:**
- Ввод симптомов пациента через формы
- Отображение результатов диагностики
- Визуализация рекомендаций
- История предыдущих консультаций

**Технологии:** HTML5, CSS3, JavaScript (React)

---

#### 2. МАШИНА ВЫВОДА (Inference Engine)

**Функции:**
- Прямой вывод (от фактов к заключению)
- Обратный вывод (проверка гипотез)
- Обработка правил из базы знаний
- Вычисление факторов уверенности
- Разрешение конфликтов между правилами

**Алгоритм работы:**
```
1. Получить факты из рабочей памяти
2. Для каждого правила:
   - Проверить условия IF
   - Если выполняются → активировать правило
3. Выбрать правило с максимальным CF
4. Применить заключение THEN
5. Добавить результат в рабочую память
```

---

#### 3. БАЗА ЗНАНИЙ

**Содержание:**
- Онтология (классы, атрибуты, связи)
- 12 продукционных правил
- Факты о заболеваниях
- Справочная медицинская информация

**Формат хранения:** JSON/XML для онтологии, правила в формате IF-THEN

---

#### 4. РАБОЧАЯ ПАМЯТЬ

**Содержание:**
- Текущие данные пациента
- Введенные симптомы
- Активированные правила
- Промежуточные выводы
- Цепочка рассуждений

**Структура данных:**
```
{
  "patient": {
    "age": 35,
    "temperature": 39.2,
    "symptoms": ["кашель", "слабость", "ломота"]
  },
  "activated_rules": ["R001", "R008"],
  "conclusions": [
    {"diagnosis": "Грипп А", "confidence": 0.85}
  ]
}
```

---

#### 5. МОДУЛЬ ОБЪЯСНЕНИЙ

**Функции:**
- Объяснение хода рассуждений
- Обоснование диагноза
- Визуализация цепочки правил
- Ответы на вопросы "Почему?"

**Пример объяснения:**
```
Диагноз: Грипп типа A (85% уверенность)

Обоснование:
1. Активировано правило R001
2. Совпали условия:
   ✓ Температура 39.2°C (>38.5°C)
   ✓ Резкое начало
   ✓ Сильная ломота в мышцах
   ✓ Выраженная слабость
3. Альтернативные диагнозы:
   - ОРВИ (исключено из-за резкого начала)
   - COVID-19 (нет потери обоняния)
```

---

#### 6. МОДУЛЬ РЕКОМЕНДАЦИЙ

**Функции:**
- Формирование плана лечения
- Выдача медикаментозных рекомендаций
- Определение необходимости обращения к врачу
- Рекомендации по режиму

---

### 3.2. Взаимодействие компонентов

```
[Пользователь]
      ↓
[Пользовательский интерфейс]
      ↓
[Рабочая память] ←→ [Машина вывода] ←→ [База знаний]
      ↓                    ↓
[Модуль объяснений]  [Модуль рекомендаций]
      ↓                    ↓
[Пользовательский интерфейс]
      ↓
[Пользователь]
```

---

### 3.3. Процесс принятия решений

**Шаг 1: Сбор данных**
- Ввод симптомов через интерфейс
- Сохранение в рабочую память

**Шаг 2: Активация правил**
- Машина вывода проверяет условия всех правил
- Формируется список подходящих правил

**Шаг 3: Разрешение конфликтов**
- Выбор правила с максимальным CF
- Учет специфичности диагноза
- Приоритет опасных состояний

**Шаг 4: Формирование вывода**
- Установление диагноза
- Вычисление итогового CF
- Генерация рекомендаций

**Шаг 5: Объяснение**
- Формирование обоснования
- Визуализация цепочки правил
