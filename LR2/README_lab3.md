# Экспертная система диагностики простудных заболеваний
## Прототип ядра ЭС - Практическая работа №3

Работающий прототип экспертной системы на основе библиотеки **experta** (Python).

---

## Описание

Экспертная система для автоматизированной диагностики распространенных простудных заболеваний на основе введенных симптомов.

**Реализованные возможности:**
- 7 продукционных правил диагностики
- Механизм вывода с цепочками рассуждений
- Блок объяснений (трассировка сработавших правил)
- Вычисление факторов уверенности (CF)
- Интерактивный ввод через консоль
- Загрузка тестовых сценариев из JSON
- Генерация рекомендаций с приоритетами

## Режимы работы

### Режим 1: Интерактивный ввод

Система задаст серию вопросов о:
- Возрасте пациента
- Наличии хронических заболеваний
- Температуре тела
- Характере начала (резкое/постепенное)
- Респираторных симптомах (насморк, кашель, боль в горле)
- Общих симптомах (слабость, ломота в мышцах)
- Специфических симптомах (потеря обоняния/вкуса)
- Критических симптомах (тяжелая одышка, низкая сатурация)

**Пример диалога:**
```
Возраст пациента: 35
Есть ли хронические заболевания? (да/нет): нет
Температура тела (°C): 38.1
Характер начала (резкое/постепенное): постепенное
Есть насморк? (да/нет): да
...
```

### Режим 2-4: Тестовые сценарии

Автоматически загружают предустановленные данные из JSON файлов.

**Сценарий #1:** Грипп у пожилого пациента  
**Сценарий #2:** Подозрение на COVID-19  
**Сценарий #3:** Критическое состояние (пневмония с дыхательной недостаточностью)

## Архитектура

### Классы фактов

```python
class Patient(Fact):        # Данные пациента
class Symptom(Fact):        # Симптомы
class Diagnosis(Fact):      # Диагноз
class RiskLevel(Fact):      # Уровень риска
class Recommendation(Fact): # Рекомендации
```

### Движок вывода

```python
class MedicalDiagnosisEngine(KnowledgeEngine):
    # Прямой вывод (forward chaining)
    # Автоматическая активация правил при добавлении фактов
```

### Правила

Определяются через декоратор `@Rule`:

```python
@Rule(
    Symptom(name='температура', value=P(lambda x: x > 38.5)),
    Symptom(name='начало', value='резкое'),
    # ... другие условия
)
def rule_001_flu_type_a(self):
    # Действия при срабатывании
    self.declare(Diagnosis(...))
    self.declare(Recommendation(...))
```

## Объяснение работы

### Как работает механизм вывода?

1. **Декларация фактов:**
```python
engine.declare(Patient(age=35))
engine.declare(Symptom(name='температура', value=39.2))
```

2. **Автоматическая активация правил:**
- Движок проверяет условия всех правил
- Если условия правила выполнены → правило срабатывает
- Правило добавляет новые факты (диагноз, рекомендации)

3. **Цепочки вывода:**
- Новые факты могут активировать другие правила
- Например: Грипп (R001) → Высокий риск у пожилых (R008)

4. **Объяснения:**
- Каждое правило записывает свое срабатывание
- Результат: полная трассировка рассуждений

## Особенности реализации

### Использование предикатов

```python
# Проверка диапазона
Symptom(name='температура', value=P(lambda x: x > 38.5))

# Проверка любого значения
Symptom(name='одышка', value=MATCH.any_value)
```

### Логические операторы

```python
# ИЛИ
OR(
    Symptom(name='потеря_обоняния', value=True),
    Symptom(name='потеря_вкуса', value=True)
)

# И
AND(
    Symptom(name='температура', value=P(lambda x: x > 40)),
    Symptom(name='жаропонижающие_помогают', value=False)
)

# НЕ
NOT(Diagnosis())
```

### Факторы уверенности

Каждый диагноз имеет фактор уверенности (CF):
- **1.0 (100%)** - абсолютная уверенность (критические состояния)
- **0.85-0.95** - высокая уверенность (специфические симптомы)
- **0.70-0.80** - умеренная уверенность (требуется подтверждение)

## Исправления в процессе разработки

### Исправление №1: Сосуществование диагнозов

**Проблема:** Правило R005 (Пневмония) не срабатывало вместе с R012 (Критическое состояние)

**Причина:** Условие `NOT(Diagnosis())` блокировало R005 после срабатывания R012

**Решение:** Удалено `NOT(Diagnosis())` из R005, так как пневмония может быть причиной критического состояния

### Исправление №2: Сортировка рекомендаций

**Проблема:** Рекомендации выводились в случайном порядке

**Решение:** Добавлена сортировка по приоритету (экстренный → критический → высокий → средний → низкий)
