# Вело Predict — прогнозирование спроса на велопрокат

Командный проект по дисциплине "Интеллектуальные системы" (ПГНИУ, факультет ИКНТ, группа ИТ-34).

**Команда:** Коньшин Никита, Мельникова Софья, Зырянова Ника, Резепин Антон.

## Задача

Регрессия: по погодным условиям, времени суток и дню недели предсказать количество велопрокатов за час.

## Датасет

**UCI Bike Sharing Dataset** — данные системы Capital Bikeshare (Вашингтон, 2011-2012).

- Источник: [UCI ML Repository](https://archive.ics.uci.edu/dataset/275/bike+sharing+dataset)
- Файл: `bike_data/hour.csv` — 17,379 записей
- Целевая переменная: `cnt` (количество прокатов за час)
- Признаки: сезон, месяц, час, день недели, погода, температура, влажность, ветер

## Установка и запуск

```bash
pip install -r requirements.txt

# Обучение модели
python train.py

# Запуск веб-интерфейса
python app.py

# Запуск тестов
pytest tests/ -v
```

## Результаты

Сравнение трёх принципиально разных подходов на тестовой выборке:

| Модель | MAE | RMSE | R2 |
|--------|-----|------|----|
| Baseline (среднее за час, своя реализация) | 87.34 | 126.83 | 0.4920 |
| **RandomForest** | **32.67** | **51.48** | **0.9163** |
| GradientBoosting | 34.86 | 51.92 | 0.9149 |

Лучшая модель: **RandomForest** — объясняет 91.6% дисперсии спроса, средняя ошибка ~33 проката/час. Сложные модели работают в 2.7 раза точнее baseline-подхода.
