## Теоретическая база

### SMOTE (Synthetic Minority Over-sampling Technique)

Метод борьбы с дисбалансом классов. Вместо простого копирования редких объектов **генерирует синтетические** — берёт два близких объекта миноритарного класса и создаёт новый на отрезке между ними в пространстве признаков. Это даёт модели больше "разнообразных" примеров редкого класса.

Ключевое правило: SMOTE применяется **только к тренировочным данным**. Если применить к тесту — мы будем оценивать модель на искусственных данных, что обесценивает оценку.

### Кривые обучения (Learning Curves)

Графики зависимости метрики (F1) от размера обучающей выборки. Строятся две кривые:
- **Train** — качество на обучающих данных
- **Validation** — качество на валидации (кросс-валидация)

Диагностика:
- **Большой разрыв** (train высоко, val низко) = переобучение. Модель выучила шум в данных. Решение: упростить модель или добавить данных.
- **Обе кривые низко** = недообучение. Модель слишком простая. Решение: усложнить модель.
- **Кривые близко и высоко** = хорошая модель. Мало пространства для улучшения.

### Pipeline

Цепочка шагов обработки данных. Объединяет масштабирование и модель в один объект. Это:
1. Гарантирует, что scaler.fit() вызывается только на train-фолде (нет утечки данных)
2. Упрощает код — один .fit() вместо нескольких
3. Необходим для корректной работы с GridSearchCV

### ImbPipeline (из imblearn)

Расширенная версия Pipeline, которая умеет работать с ресэмплерами (SMOTE). В обычном sklearn Pipeline нельзя вставить шаг, который меняет количество строк. ImbPipeline позволяет включить SMOTE как шаг пайплайна, и при кросс-валидации SMOTE будет применяться **только к тренировочному фолду** каждой итерации.

### GridSearchCV

Полный перебор комбинаций гиперпараметров с кросс-валидацией. Для каждой комбинации:
1. Данные делятся на k фолдов (у нас k=3)
2. k раз модель обучается на k-1 фолдах и проверяется на оставшемся
3. Среднее качество по k итерациям — оценка этой комбинации

Выбирается комбинация с лучшим средним качеством.

### StratifiedKFold

Вариант k-fold, где в каждом фолде сохраняются пропорции классов. Без стратификации при маленьком k (3) в фолде может не оказаться объектов редкого класса.

### Гиперпараметры RandomForest

- **n_estimators** — количество деревьев в ансамбле. Больше деревьев = стабильнее предсказание, но дольше обучение.
- **max_depth** — максимальная глубина дерева. Ограничивает сложность. None = без ограничения (дерево растёт пока может).
- **min_samples_leaf** — минимальное число объектов в листе. Больше = проще дерево, меньше переобучение.

### Шаг 3: GridSearchCV (строки 188-252)

```python

        ('scaler', StandardScaler()),     # Шаг 1: масштабирование

        ('scaler', StandardScaler()),
```

## Вопросы

1. **Что такое SMOTE и зачем он?** — Генерирует синтетические объекты между реальными точками миноритарного класса. Нужен для борьбы с дисбалансом — чтобы модель не "забывала" про редкие классы.

2. **Почему SMOTE только на train?** — Если применить к тесту, мы оцениваем модель на ненастоящих данных. Тест должен отражать реальное распределение.

3. **Зачем ImbPipeline, а не обычный Pipeline?** — Обычный Pipeline не может менять количество строк (SMOTE добавляет). ImbPipeline умеет: при fit вызывает fit_resample (SMOTE), а при predict/transform пропускает ресэмплинг. Это гарантирует, что SMOTE не утечёт в валидационные фолды при CV.

4. **Что показывают кривые обучения?** — Как качество зависит от размера данных. Если train сильно выше val — переобучение (модель слишком сложная). Если обе низко — недообучение (модель слишком простая).

5. **Что делает GridSearchCV?** — Перебирает все комбинации гиперпараметров. Для каждой комбинации делает k-fold кросс-валидацию, считает среднее качество. Выбирает лучшую комбинацию.

6. **Зачем StratifiedKFold?** — Чтобы в каждом фолде были объекты всех классов в правильных пропорциях. Без стратификации при k=3 и маленьком классе может получиться фолд без объектов одного класса.

7. **Что значит rf__ в param_grid?** — Префикс шага в Pipeline. Pipeline — цепочка именованных шагов. `rf__max_depth` означает "параметр max_depth шага rf". Двойное подчёркивание — разделитель.

8. **Почему на Wine улучшение 0%?** — Датасет Wine очень хорошо разделим для RandomForest — базовая модель уже даёт F1 = 1.0. Улучшать некуда. На более сложных датасетах (например, с перекрывающимися классами) разница была бы заметной.

9. **Чем feature_importances_ полезны?** — Показывают, какие признаки модель считает самыми важными для предсказания. Помогает понять модель, отсеять ненужные признаки и выявить domain-инсайты.
