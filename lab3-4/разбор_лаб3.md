---

## Теоретическая база

### Мультиклассовая классификация

Задача, где целевая переменная принимает больше двух значений. В нашем случае — 3 сорта винограда. Модели должны уметь различать все три класса одновременно.

### Метрики

- **Accuracy** — доля правильных ответов от общего числа. Простая, но может вводить в заблуждение при дисбалансе классов (если один класс 90%, модель может просто всегда предсказывать его и получить 90%).
- **Precision** (точность) — из всех предсказаний "класс X", сколько действительно были классом X. Отвечает на вопрос: "Если модель сказала X — насколько ей можно верить?"
- **Recall** (полнота) — из всех реальных объектов класса X, сколько модель нашла. Отвечает на вопрос: "Не пропускает ли модель объекты этого класса?"
- **F1-score** — гармоническое среднее precision и recall. Балансирует между ними. F1 = 2 * (P * R) / (P + R).
- **Macro F1** — F1 считается отдельно для каждого класса, потом берётся среднее. Все классы равнозначны, не важно сколько объектов в каждом. Именно эту метрику мы сравниваем.

### Confusion Matrix (матрица ошибок)

Таблица NxN (где N — число классов), показывающая куда модель "путает" объекты. По диагонали — правильные ответы, вне диагонали — ошибки. Позволяет увидеть, какие конкретно классы модель путает между собой.

### class_weight='balanced'

Параметр RandomForest, который автоматически увеличивает штраф за ошибки на редких классах. Веса обратно пропорциональны частоте класса: w_i = n_samples / (n_classes * n_samples_i). Помогает модели не "забывать" про малочисленные классы.

### Стратификация (stratify)

При разделении на train/test сохраняет пропорции классов в обеих выборках. Без неё может случиться, что в тест попадёт мало объектов редкого класса — и оценка будет ненадёжной.

### StandardScaler

Приводит каждый признак к среднему=0 и стандартному отклонению=1: z = (x - mean) / std. RandomForest не требует масштабирования (работает с порогами), а LogisticRegression — требует, потому что использует градиентный спуск и чувствительна к масштабу признаков.

## Вопросы

1. **Зачем stratify при split?** — Чтобы сохранить пропорции классов. Без неё в маленьком тесте может не оказаться объектов редкого класса.

2. **Почему LR масштабируем, а RF нет?** — LR использует градиентный спуск и чувствительна к масштабу. RF работает с порогами на признаках — масштаб не важен.

3. **Что такое macro F1 и почему именно он?** — Среднее F1 по классам, каждый класс одинаково важен. При дисбалансе accuracy может быть высокой за счёт мажорного класса, а macro F1 покажет проблему.

4. **Как работает class_weight='balanced'?** — Увеличивает штраф за ошибки на редких классах пропорционально их редкости. Формула: w = n / (k * n_i).

5. **Что показывает матрица ошибок?** — Какие классы модель путает. Строка i, столбец j — сколько объектов класса i предсказано как j. Диагональ — верно, остальное — ошибки.

6. **Почему fit только на train?** — Чтобы не было утечки данных (data leakage). Модель и scaler не должны "видеть" тест до финальной оценки.
